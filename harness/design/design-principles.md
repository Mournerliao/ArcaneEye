# ArcaneEye 设计原则

## 核心设计原则

### 1. 专用化原则
- **专注于大乱斗场景**：只关注选人阶段和海克斯选择阶段
- **极致打磨**：在 20-30 秒内提供最有价值的信息
- **避免功能蔓延**：不添加与核心场景无关的功能

### 2. 用户体验原则
- **无干扰**：HUD 面板必须保持鼠标可穿透，不夺取游戏窗口焦点
- **即时反馈**：截图后立即显示识别结果和推荐
- **自动隐藏**：5 秒后自动隐藏，不干扰游戏进程
- **高亮锁定**：最佳选项用发光魔法符文圈高亮显示

### 3. 数据驱动原则
- **实时数据**：使用最新的 ARAMGG 数据 API
- **数据准确性**：确保胜率数据的准确性和时效性
- **RAG 增强**：结合实时数据生成推荐，提升准确度

### 4. 技术架构原则
- **模块化设计**：截图功能作为可插拔模块，支持多种截图后端
- **开箱即用**：内置基本截图能力，无需额外配置
- **可扩展性**：支持集成第三方 HDR 截图工具作为增强方案

## UI/UX 设计规范

### 视觉设计
- **CSS 框架**：Tailwind CSS
- **UI 组件库**：shadcn/ui
- **设计系统**：待定（需要详细设计）

### 交互设计
- **触发方式**：全局快捷键 (Alt + Q)
- **反馈方式**：HUD 悬浮窗特效滑出
- **消失方式**：5 秒倒计时自动隐藏

### 无障碍设计
- **高对比度**：确保在游戏背景下清晰可见
- **可读性**：字体大小和颜色易于阅读
- **响应性**：适应不同屏幕尺寸和分辨率

## 数据设计原则

### 数据源
- **主要数据源**：ARAMGG 数据 API (https://data.dtodo.cn)
- **数据同步**：每天定时同步到本地数据库
- **数据存储**：Supabase PostgreSQL + pgvector

### 数据流
1. 截图识别英雄 → 触发数据查询
2. 查询 ARAMGG API → 获取最新胜率数据
3. 数据与 Prompt 拼接 → 喂给大模型
4. 生成推荐结果 → 显示在 HUD 面板

## 安全性原则

### 数据安全
- **API Key 安全**：使用环境变量存储，不提交到代码仓库
- **数据隐私**：不收集用户个人信息，只收集游戏数据
- **通信安全**：使用 HTTPS 进行 API 通信

### 系统安全
- **进程隔离**：截图模块与主应用进程分离
- **资源管理**：确定性释放系统资源
- **错误处理**：优雅处理各种异常情况
